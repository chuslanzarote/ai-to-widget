## 1. Copy the bundle onto your web server

Serve `widget.js` and `widget.css` as static assets under a path you
control, for example `/static/widget.js` and `/static/widget.css`.

```bash
cp dist/widget.js /var/www/static/widget.js
cp dist/widget.css /var/www/static/widget.css
```

## 2. Add the `<link>` and `<script>` to every page

Near the closing `</body>`:

```html
<link rel="stylesheet" href="/static/widget.css" />
<script
  src="/static/widget.js"
  defer
  data-backend-url="http://localhost:3100"
  data-api-base-url="http://localhost:3200"
  data-auth-mode="bearer"
  data-auth-token-key="shop_auth_token"
  data-launcher-position="bottom-right"
  data-locale="en-US"
  data-login-url="http://localhost:8080/login"
></script>
```

Put the tag on the layout/partial that renders site-wide so the widget
appears on every page.

## 3. CORS on your host API

- `Access-Control-Allow-Origin: <your storefront origin>`
- `Access-Control-Allow-Methods: GET, POST, PATCH, DELETE, OPTIONS`
- `Access-Control-Allow-Headers: Content-Type, Authorization`

## 4. CORS on the ATW backend

```env
ALLOWED_ORIGINS=https://your-site.example.com
```

## 5. Theming

```css
}
```

## 6. Troubleshooting

- **Launcher missing** — make sure the `<script>` tag is not inside an
  iframe or Shadow DOM; it needs top-level access to `document.body`.
- **CORS preflight failing** — the host API needs to handle `OPTIONS`
  requests explicitly; many vanilla servers default to 405.
- **401 on actions** — user is not authenticated in the host app; the
  widget will surface a "please log in first" message.
