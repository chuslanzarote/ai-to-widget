## Entity: Product

**Purpose.** Represents a coffee bag or a piece of brewing equipment for sale in the shop's catalog.

**Classification**: indexable

**Source tables**: products

**Joined references**: 

**Primary table.** `products`

### Columns

- `id`: index Stable identifier for tool calls.
- `handle`: index URL slug used by the storefront.
- `name`: index Primary display name.
- `description`: index Free-text where origin / process / roast / sensory descriptors live.
- `price_cents`: index Stored as integer cents; format at render time.
- `image_url`: index Product imagery reference.

**Fields indexed for RAG.**

| Field | Type | Included | Rationale |
|---|---|---|---|
| `id` | uuid (PK) | yes, as identifier | Needed for tool calls (e.g. add-to-cart, comparisons). |
| `handle` | text (slug) | yes | Stable URL slug; useful when the shopper references a product by its storefront link. |
| `name` | text | yes | Primary display name. |
| `description` | text | yes | Carries origin / process / roast / sensory descriptors — the heart of retrieval for this catalog. |
| `price_cents` | integer | yes, informational | Acceptable staleness for a 20-row demo catalog; the runtime should re-read price at action time. |
| `image_url` | text | yes | Lets the chat surface / reference product imagery when relevant. |
| `in_stock` | boolean | no (filter) | Used as an ingest filter: only `in_stock = true` rows produce documents. |
| `created_at` | timestamp with time zone | no | Not user-facing. |

**Related tables joined for enrichment.** None — single-table entity in this dump.

**Document assembly.** One document per `products` row. The document concatenates `name`, `handle`, `description`, a formatted price (from `price_cents`), and the `image_url`. The brief's coffee vocabulary (origen, varietal, proceso, perfil de tueste, notas de cata, método de extracción, SCA score, decaf, single origin, blend) is pattern-extracted during enrichment from the free-text `description` field; the schema does not store those concepts as separate columns.

**PII flags.** None. All columns are customer-facing catalog data; no column name or sample value matched any PII signal.

**Filters applied during ingestion.**
- `in_stock = true`

**Estimated volume.** ~20 rows after filters → ~20 documents.

## Ignored tables

None present in this dump. The wider host database contains `carts`, `cart_items`, `orders`, `order_items`, and `users`, but they were deliberately excluded from the schema input on the Builder's instruction: those entities are accessed at runtime through host-API actions (defined in `/atw.api`) rather than indexed for retrieval. This separation prevents one shopper's transactional data from leaking into another shopper's RAG result.
