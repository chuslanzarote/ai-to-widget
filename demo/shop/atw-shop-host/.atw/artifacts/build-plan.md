## Enrichment plan

### Entity: Product (20 docs)

- **Source.** `client_ref.products` rows where `in_stock = true`.
- **Document template.** Concatenate `name` + `handle` + `description` + formatted `price_cents` + `image_url` (per `schema-map.md` § Document assembly).
- **Categories to derive (vocabulary anchored to brief.md § Business vocabulary).** Pattern-extracted from `description` text:
  - `origin` (country / region / farm)
  - `varietal` (e.g. Geisha, Bourbon, Typica)
  - `process` (washed, natural, honey, anaerobic)
  - `roast_profile` (light / medium / dark)
  - `tasting_notes` (floral, citric, chocolate, red fruit, etc.)
  - `brew_method_fit` (V60, espresso, AeroPress, Chemex, French press)
  - `decaf_method` (Swiss Water, CO2, EA — when applicable)
  - `single_origin_or_blend`
- **Equipment-only items.** The catalog mixes coffee + brewing equipment (kettles, scales, mugs, grinders, cleaning tablets); enrichment must not collapse equipment into coffee vocabulary (per Builder's override in `schema-map.md`). For non-coffee items the categories above resolve to `null` and the `material` / `function` axis is captured in free-text.
- **Enrichment LLM.** `claude-opus-4-7` (model snapshot pinned in `project.md`).
- **Calls.** 20 total (one per product), 1× multiplier.

## Build sequence

1. **IMPORT.** Apply migrations, import `.atw/inputs/schema.sql` into `client_ref.products`.
2. **ENRICH.** Run 20 enrichment LLM calls (one per product), populate `atw_documents`.
3. **EMBED.** Generate 384-dim vectors for each enriched doc, write to `embedding`.
4. **RENDER.** Generate the backend code (action executors from `action-manifest.md`, system prompt from `brief.md` + `schema-map.md`).
5. **COMPOSE.** Activate the host `docker-compose.yml` (or warn if marker block absent).
