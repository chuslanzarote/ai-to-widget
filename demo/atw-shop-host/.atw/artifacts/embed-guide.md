Target host: the reference shop SPA under `demo/shop/frontend` (Vite + React,

## 1. Copy the bundle into the host's static root

From the Builder's working directory (this repo), the compiled widget lives at
`dist/widget.js` + `dist/widget.css`. Copy both into the shop frontend's
public root so nginx serves them alongside the SPA:

```bash
cp dist/widget.js  ../shop/frontend/public/widget.js
cp dist/widget.css ../shop/frontend/public/widget.css
```

The bundle is deterministic (Feature 002 determinism contract, FR-022), so
committing the copied files — or re-running `/atw.build` on demand — is safe.

## 2. Add the `<link>` + `<script>` pair to `index.html`

Edit `demo/shop/frontend/index.html`, just before `</body>`:

```html
<link rel="stylesheet" href="/widget.css" />
<script
  src="/widget.js"
  defer
  data-backend-url="http://localhost:3100"
  data-api-base-url="http://localhost:3200"
  data-auth-mode="bearer"
  data-bearer-storage-key="shop_auth_token"
  data-login-url="http://localhost:8080/login"
  data-launcher-position="bottom-right"
  data-locale="en-US"
></script>
```

Why these values:

| Attribute                  | Value                            | Source                                           |
|----------------------------|----------------------------------|--------------------------------------------------|
| `data-backend-url`         | `http://localhost:3100`          | `atw_backend` port in `docker-compose.yml`       |
| `data-api-base-url`        | `http://localhost:3200`          | `shop_backend` port in `demo/shop/docker-compose.yml` |
| `data-auth-mode`           | `bearer`                         | FR-006 — shopper JWT, set by the shop SPA        |
| `data-bearer-storage-key`  | `shop_auth_token`                | `demo/shop/frontend/src/auth/token.ts`           |
| `data-login-url`           | `http://localhost:8080/login`    | shop SPA login route                             |

## 3. CORS on the shop backend (host API)

The widget calls the shop API directly from the browser with the shopper's
bearer token (Feature 007 red line — `atw_backend` never reaches the shop).
The shop backend must accept the storefront origin:

- `Access-Control-Allow-Origin: http://localhost:8080`
- `Access-Control-Allow-Methods: GET, POST, PATCH, DELETE, OPTIONS`
- `Access-Control-Allow-Headers: Content-Type, Authorization`
- `Access-Control-Expose-Headers: Content-Type`

Credentials are **not** required — bearer tokens travel in the `Authorization`
header, not cookies. Do not set `Access-Control-Allow-Credentials: true` and
do not use `Access-Control-Allow-Origin: *` together with credentials.

## 4. CORS on the ATW backend

`atw_backend` reads `ALLOWED_ORIGINS` from its environment. The root
`docker-compose.yml` already sets:

```yaml
ALLOWED_ORIGINS: http://localhost:8000,http://localhost:8080
```

`http://localhost:8080` is the shop storefront — keep it in the list. Add any
additional origin where the storefront will be served (comma-separated, no
trailing slashes, exact scheme + host + port match).

## 5. Theming

The widget exposes CSS custom properties on its root element. Override them
from the shop frontend's stylesheet (e.g., `demo/shop/frontend/src/index.css`)
to match the host theme. No overrides were supplied in the embed answers, so
the widget renders with its built-in defaults:

```css
}
```

## 6. Troubleshooting

- **Launcher does not appear** — open DevTools → console and look for an
  `[atw]` error. The most common cause is a missing or malformed
  `data-backend-url`, or the shop SPA not being rebuilt after copying the
  bundle into `public/`.
- **CORS error from `atw_backend`** — origin rejected. Confirm the browser's
  address bar matches an entry in `ALLOWED_ORIGINS` exactly (scheme, host,
  port — no trailing slash).
- **CORS error from the shop backend** — the shop API rejected the widget's
  direct call. Check the shop backend's CORS config allows the storefront
  origin and both `Content-Type` and `Authorization` request headers.
- **401 on tool calls** — the shopper is not signed in, or the bearer token
  has expired. The widget reads `localStorage['shop_auth_token']`; if that key
  is missing or stale, the widget should surface its anonymous fallback and
  link to `http://localhost:8080/login`.
- **`ATW_TOOL_NOT_ALLOWED` in the console** — the backend tried to emit a
  tool call outside the widget allowlist. This is a defence-in-depth refusal;
  inspect `.atw/artifacts/action-manifest.md` to see which tools are
  authorised.
- **`atw_backend` making outbound calls to the shop** — Feature 007 red line
  violation. Verify no `HOST_API_BASE_URL` / `HOST_API_KEY` envs are set on
  `atw_backend` and re-read `specs/007-widget-tool-loop/spec.md` US5.
